# Glyph Compare (字形相似度比对)

基于图像特征的轻量级字形相似度比对系统。本项目**不依赖于传统的 OCR 识别模型**，而是纯粹从计算机视觉和数字图像处理的角度，提取字形的结构、轮廓以及笔画密度分布特征，通过多算法加权融合打分，实现快速、精准的字形相似度比对。

---

## 1. 项目功能

* **单对单字形比对**：比较两张字形图片之间的相似度，输出各项算法的得分以及综合相似度。
* **批量与目录比对**：支持一张参考图与多张候选图或指定目录下的所有字形图片进行批量比对。
* **字形动态渲染**：支持指定字符（如某个汉字）以及字体文件路径（支持**本地绝对路径/相对路径**或 **Classpath 类路径资源**，如 `.ttf` 格式），动态将字形渲染为标准图片作为参考图进行比对。
* **结果智能过滤与截取**：
  * **阈值判断**：自动判定字形是否相同（默认综合评分 $\ge 80$ 分判定为 `SAME`）。
  * **自动筛选**：比对结果中**仅保留判定为 `SAME` 的匹配项**，并按相似度从高到低排序，**最多只保留前 5 个最相似的结果**，避免输出大量无关的低相似度结果。

## 3. 实现思路与算法

本项目为了实现对字形的高容错比对，采用了**预处理 + 多维特征提取 + 加权融合**的思路。

### 3.1 图像预处理 (Image Preprocessor)
直接对整张图片进行比对会受到文字所处位置、图片留白大小的极大干扰。因此，在比对前，`ImagePreprocessor` 会执行以下操作：
1. **二值化**：将图片转为灰度图并进行二值化，使字形呈黑色，背景呈白色。
2. **最小外接矩形裁剪**：寻找字形像素的非零区域，精准计算字形的最小外接矩形并进行裁剪，**彻底消除字形在图片中的位置偏移和空白边距干扰**。
3. **尺寸归一化**：将裁剪后的字形区域缩放到固定大小（例如 $64 \times 64$ 像素），确保不同分辨率 of 图片具有可比性。

### 3.2 核心比对算法 (Algorithms)
字形差异通常体现在笔画粗细、字体局部变形、笔画宏观结构等方面。为此，系统采用了三种互补的算法进行多角度评估：

1. **网格直方图算法 (Grid Histogram Algorithm) - 权重 40%**
   * **原理**：将归一化后的字形划分为多个网格区域（如 $8 \times 8$ 的网格），计算每个网格内黑色像素（笔画）的密度分布，最后计算两张图密度的相关性。
   * **作用**：能够很好地反映笔画在空间上的粗细分布，对局部微小的噪点不敏感，有效对抗笔画粗细差异。

2. **轮廓匹配算法 (Contour Match Algorithm) - 权重 30%**
   * **原理**：提取字形的最大外部轮廓，利用 OpenCV 的 `matchShapes` 计算两者的 **Hu 不变矩 (Hu Moments)**。
   * **作用**：Hu 不变矩具有平移、旋转和尺度不变性，能从宏观的几何拓扑结构层面评估字形轮廓的相似度。

3. **距离容忍匹配算法 (Distance Tolerant Algorithm) - 权重 30%**
   * **原理**：逐像素对比两张图。当参考图的某个笔画像素在候选图的对应位置找不到匹配时，允许其在邻域（如周围 1~2 像素范围）内寻找匹配像素，若在邻域内找到则给予一定的“距离折损得分”。
   * **作用**：对边缘抗锯齿、渲染产生的微小位移和形变具有极高的容忍度。

### 3.3 综合评分机制
最终综合评分为三种算法得分的加权和：
$$\text{TotalScore} = \text{GridScore} \times 0.4 + \text{ContourScore} \times 0.3 + \text{DistanceScore} \times 0.3$$
当 $\text{TotalScore} \ge 80.0$ 时，判定两个字形为相同字形（`SAME`）。

## 4. 实现技术

* **开发语言**：Java 17
* **图形与渲染**：Java AWT / Swing (`Graphics2D`、`BufferedImage`)。用于实现读取 `.ttf` 字体文件，并将任意字符高保真地渲染为标准图片，为比对提供无失真的参考源。
* **计算机视觉库**：**OpenCV 4.9.0** (使用 `org.openpnp:opencv` 自主封装依赖)。
  * `openpnp-opencv` 极大简化了 Java 使用 OpenCV 的门槛，**无需手动配置本机的 `dll` 或 `so` 动态链接库路径**，在项目启动时通过 `OpenCV.loadLocally()` 即可自动解压并加载对应操作系统的本地库，支持 Windows、macOS 和 Linux。
* **构建工具**：Maven 3

## 5. 达到的目的

1. **摆脱 OCR 的局限性**：无需部署庞大繁重的深度学习模型或调用第三方 OCR API，在一台普通的 CPU 机器上即可以毫秒级速度完成字形特征的比对。
2. **高鲁棒性**：通过外接矩形裁剪、网格密度分析以及距离折损算法，有效解决了图片背景差异、边距大小不一、字体笔画粗细不等、渲染锯齿和微小抖动对图像比对造成的假阴性误差。
3. **应用场景广泛**：
   * **字体识别与比对**：判断某张汉字图片是由哪种字体渲染出来的。
   * **古籍/生僻字整理**：在无法进行 OCR 的场景下，基于形状特征对生僻字、异体字进行聚类或查找。
   * **文字纠错**：比对印刷体或排版字形与标准字形的结构偏差。
